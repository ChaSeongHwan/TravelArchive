# 사용자 DB 설계 초안 (v0)

## PostgreSQL 테이블 형상

### `users` — 루트 식별자

| 칼럼명 | 타입 | NULL | 키 | 기본값 | 설명 |
|--------|------|------|----|--------|------|
| user_id | VARCHAR(40) | NO | PK | — | `MEM:uuid`, `KKO:hash` 등 |
| user_type | VARCHAR(3) | NO | — | — | `MEM` `GST` `KKO` `NVR` `GGL` |
| status | VARCHAR(10) | NO | — | `active` | `active` `inactive` `banned` |
| created_at | TIMESTAMPTZ | NO | — | NOW() | 계정 생성 시각 |

## Redis 키 형상 (TTL 포함)

| 키 패턴 | 저장 값 | TTL | 대상 |
|---------|---------|-----|------|
| `auth:refresh:{jti}` | user_id | 7일 | 전체 |
| `user:GST:{uuid}` | 게스트 기본 정보 JSON | 24h | GST만 |
| `session:{session_id}:meta` | 세션 메타 JSON | MEM 48h / GST 24h | 전체 |
| `session:{session_id}:state` | `idle` `processing` `error` | MEM 48h / GST 24h | 전체 |
| `user:{user_id}:sessions` | session_id Set | MEM 48h / GST 24h | 전체 |
| `queue:tasks` | task payload List | TTL 없음 (Queue) | 전체 |

## 관계도 요약

```
users (1)
  ├── user_profile    (1:1)
  ├── user_security   (1:1)
  ├── user_oauth      (1:N)  ← SNS 복수 연동
  ├── user_preference (1:1)
  └── sessions        (1:N)
```

> `GST`는 Redis에서만 살고 PostgreSQL에는 `users` 행 자체가 없음.
> 창 닫기(`beforeunload`) 등의 트리거 발생 시 user_id prefix를 확인하여
> MEM/SNS 계열은 PostgreSQL에 flush, GST는 Redis 키 즉시 파기.
